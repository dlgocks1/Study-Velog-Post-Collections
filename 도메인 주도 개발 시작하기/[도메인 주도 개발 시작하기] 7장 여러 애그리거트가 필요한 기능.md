## 7.1 여러 애그리거트가 필요한 기능

도메인 영역의 코드를 작성하다 보면 한 애그리거트로 기능을 구현할 수 없을 때가 있다. 

![](https://velog.velcdn.com/images/cksgodl/post/72c3ddf2-b37c-4190-b2cc-143b678d6c48/image.png)

한 애그리거트에 넣기 애매한 도메인 로직은 **도메인 서비스(Domain Service)** 로 분리
도메인 서비스는 도메인 영역에 위치하며 여러 애그리거트 협력

![](https://velog.velcdn.com/images/cksgodl/post/ed7ff3d4-db6a-481b-8755-cc266e1b73b0/image.png)

## 7.2 도메인 서비스

도메인 서비스의 특징

- 상태(state)를 가지지 않음 — 로직만 제공
- 도메인 영역의 다른 구성 요소(애그리거트, 값 객체)와 동급
- 응용 서비스가 직접 호출하거나, 애그리거트의 메서드에 파라미터로 전달

```kotlin
// 도메인 서비스 - 도메인 패키지에 위치
package com.myshop.order.domain

class DiscountCalculationService {
    
    fun calculateDiscountAmount(
        orderLines: List<OrderLine>,
        coupons: List<Coupon>,
        memberGrade: MemberGrade
    ): Money {
        val couponDiscount = coupons
            .map { calculateDiscount(it, orderLines) }
            .fold(Money.ZERO) { acc, m -> acc.plus(m) }
        
        val membershipDiscount = calculateDiscount(memberGrade, orderLines)
        
        return couponDiscount.plus(membershipDiscount)
    }
    
    private fun calculateDiscount(coupon: Coupon, orderLines: List<OrderLine>): Money { /* ... */ }
    private fun calculateDiscount(grade: MemberGrade, orderLines: List<OrderLine>): Money { /* ... */ }
}
```

> 도메인 서비스 이름 짓기
>
- "서비스"라는 접미어보다 도메인 개념을 드러내는 이름이 더 좋음
   - 좋은 예: DiscountCalculationService, MoneyTransferService, SurveyPermissionChecker
도메인 전문가가 사용하는 유비쿼터스 언어를 반영

### 핵심 원칙

도메인 서비스는 도메인 영역에 위치시켜야 함
응용 영역이나 인프라 영역에 두면 도메인 로직이 흩어짐

![](https://velog.velcdn.com/images/cksgodl/post/c6444e5e-36b1-4a8c-988b-b22824c43dea/image.png)

#### 도메인 서비스가 특별한 의존을 가질 때

도메인 서비스가 외부 시스템에 의존하는 경우
- 예: 권한 시스템, 환율 시스템 등
- 인터페이스는 도메인 영역에, 구현은 인프라 영역에 분리 (의존 역전 원칙)

```kotlin
// ❌ 도메인 서비스가 직접 외부 라이브러리에 의존
package com.myshop.member.domain

class SurveyPermissionChecker(
    private val authRestTemplate: RestTemplate  // 인프라 기술에 의존
) {
    fun hasUserCreationPermission(userId: String): Boolean {
        val response = authRestTemplate.getForObject(...)
        return response.allowed
    }
}
```

```kotlin
// ✅ 도메인 영역: 인터페이스만 정의
package com.myshop.member.domain

interface SurveyPermissionChecker {
    fun hasUserCreationPermission(userId: String): Boolean
}

// ✅ 인프라 영역: 도메인 인터페이스 구현
package com.myshop.infra.auth

@Component
class HttpSurveyPermissionChecker(
    private val authClient: AuthClient
) : SurveyPermissionChecker {
    override fun hasUserCreationPermission(userId: String): Boolean {
        return authClient.check(userId, "SURVEY_CREATE")
    }
}
```
